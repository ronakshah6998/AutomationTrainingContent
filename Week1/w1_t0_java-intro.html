What is Java?

A platform-independent, object-oriented language — used everywhere from Android apps to enterprise systems.

🌐

Write Once, Run Anywhere

Java code compiles to bytecode — not machine code. The JVM runs it on any OS without recompiling.

🏗

Object-Oriented

Everything lives inside a class. Java enforces structure, making large codebases manageable.

🛡

Strongly Typed

Every variable has a declared type. No guessing — errors are caught at compile time, not runtime.

🔋

Massive Ecosystem

Maven, Spring, Selenium, JUnit — thousands of battle-tested libraries ready to use.

💡
Why Java for Automation?
Selenium WebDriver's primary bindings are in Java. Most enterprise automation frameworks (TestNG, Cucumber-JVM, RestAssured) are Java-first. Learning Java = unlocking the full power of the automation ecosystem.
The Java Family
☕

JDK — Java Development Kit

What YOU install. Includes the compiler (javac), runtime, and developer tools.

🖥

JRE — Java Runtime Env

What end-users install to run Java apps. No compiler — only the JVM + libraries.

⚙

JVM — Java Virtual Machine

The engine that executes bytecode. Each OS has its own JVM — but bytecode is the same everywhere.

Install the JDK First

Before touching any IDE, you need the Java Development Kit on your machine.

  • 1

    Download JDK 17 (LTS)

    Go to adoptium.net → choose Temurin 17 (LTS) → pick your OS. LTS = Long Term Support = stable for production.

  • 2

    Run the Installer

    Windows: double-click the .msi. Mac: open the .pkg. Accept defaults — it installs to C:\Program Files\Java\ on Windows.

  • 3

    Set JAVA_HOME (Windows)

    System → Environment Variables → New System Variable: JAVA_HOME = C:\Program Files\Java\jdk-17 then add %JAVA_HOME%\bin to PATH.

  • 4

    Verify in Terminal

    Open CMD / Terminal and run:
    java -version → should print openjdk version "17.x.x"

⚠
Tip: Always use a LTS version for learning and project work. Java 17 or Java 21 are your best bets in 2024+.

Hello World — Your First Java Program

Every journey starts here. Let's write it, understand every line, then run it.

☕ HelloWorld.java
// File must be named HelloWorld.java

public class HelloWorld {

    public static void main(String[] args) {

        System.out.println("Hello, World!");

    }

}
Line-by-Line Breakdown

Click any row to highlight what it does:

public class HelloWorld
Class declaration. In Java, every file must have a class. The class name must exactly match the filename (HelloWorld.java). public means it's accessible from anywhere.
public static void main
Entry point. This is where Java starts executing. static = no object needed; void = returns nothing; main = the JVM looks for this exact method signature.
String[] args
Command-line arguments. An array of Strings you can pass when running the program. Not used in Hello World, but always required in the signature.
System.out.println(...)
Prints to the console. System is a built-in class; out is an output stream; println prints and adds a newline. Notice the semicolon at the end — Java requires it.
"Hello, World!"
A String literal. Any text inside double quotes is a String. Java is strict about this — single quotes are for characters (char), not Strings.

🔨

javac — The Compiler

Reads your .java file and converts it to .class bytecode. Run it: javac HelloWorld.java

📦

.class — Bytecode

Platform-neutral instructions. Not readable by humans. Not tied to Windows, Mac, or Linux — the JVM handles that.

🖥

JVM — The Engine

Reads bytecode and converts it to native machine instructions in real-time (JIT compilation). Run it: java HelloWorld

🌐

Why "Write Once, Run Anywhere"?

The .class bytecode is identical on all platforms. Only the JVM differs per OS — and JetBrains/Oracle ship that for each one.

Common Errors & How to Fix Them

Every Java beginner hits these. Know them now, waste zero time later.

🔴 Error: JDK Not Configured / 'javac' is not recognized

What happened?

The system can't find the Java compiler because JAVA_HOME or PATH is not set correctly — or JDK was never installed.

'javac' is not recognized as an internal or external command, operable program or batch file.

How to Fix

1. Verify JDK is installed: C:\Program Files\Java\jdk-17 should exist.
2. Set JAVA_HOME = that path in Environment Variables.
3. Add %JAVA_HOME%\bin to the PATH variable.
4. Open a NEW terminal and run javac -version again.

✅ Fix: Set JAVA_HOME and PATH, then restart terminal
🔴 Error: Class Name Does Not Match File Name

What happened?

The class name inside the file (public class) must exactly match the filename, including case. Java is case-sensitive.

error: class helloworld is public, should be declared in a file named helloworld.java public class HelloWorld { ^ 1 error

The Mistake

File is named helloworld.java but inside you wrote public class HelloWorld — capital H, capital W.

✅ Fix: Rename file to HelloWorld.java (match exactly, case included)
🔴 Error: Missing Semicolon

What happened?

Every statement in Java must end with a semicolon ;. Forgetting it is the #1 beginner syntax error.

HelloWorld.java:5: error: ';' expected System.out.println("Hello, World!") ^ 1 error

The Mistake

Written as: System.out.println("Hello, World!") — notice the missing ; at the end.

✅ Fix: Add semicolon → System.out.println("Hello, World!");
🔴 Error: Main Method Not Found

What happened?

The JVM looks for a very specific signature: public static void main(String[] args). Any deviation means it can't find the entry point.

Error: Main method not found in class HelloWorld, please define the main method as: public static void main(String[] args)

Common Mistakes

• Writing Public instead of public (wrong case)
• Removing static
• Writing String args[] instead of String[] args (both work, but typos happen)
• Spelling it Main instead of main

✅ Fix: Copy the exact signature — public static void main(String[] args)
🔴 Runtime Error: NullPointerException

What happened?

You tried to use an object that hasn't been created yet. It's null — pointing to nothing — and you called a method on it.

Exception in thread "main" java.lang.NullPointerException: Cannot invoke "String.length()" because "str" is null at HelloWorld.main(HelloWorld.java:5)

The Mistake

String str = null; followed by str.length() — str was never assigned a real value, so Java throws NPE.

✅ Fix: Initialize the variable → String str = "Hello";
