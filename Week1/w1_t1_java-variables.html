Variables & Data Types in Java

Learn how Java stores, names, and types values — and why getting this right prevents 90% of beginner bugs.

📦 Variables 🔢 Primitives 🔤 String ⚡ Interactive
What is a Variable?

Think of a variable as a labelled box. The label is the name, the box holds a value, and the type tells Java what kind of thing fits inside.

💡
Real-world analogy: A variable is like a form field — "Age: [ 25 ]". The field name is fixed, but the value can change. Java just makes you declare what kind of value (number, text, true/false) goes in before you use it.

Primitive Data Types

Primitives hold actual values — not objects. Java has 8 primitive types; these 4 cover 95% of what you'll use.

int Integer

Stores whole numbers (no decimals). Range: −2B to +2B.

int age = 25;
Use for: counts, IDs, loops
double Decimal

Stores numbers with decimal points (floating-point).

double price = 99.99;
Use for: prices, percentages
boolean True/False

Stores only two values: true or false.

boolean isLoggedIn = true;
Use for: flags, conditions
char Single Character

Stores exactly one character. Must use single quotes.

char grade = 'A';
Use for: status codes, grades
Non-Primitive: String

String is not a primitive — it's a full class in Java. That means it comes with built-in methods you can call on it.

String Text / Sequence of Characters

Stores any text. Always wrapped in double quotes. Immutable — once created, its value doesn't change internally.

String name = "Rahul"; // "Rahul" stored as an object
Use for: names, URLs, API responses, messages
☕ String — Built-in Methods
String url = "https://api.example.com/users";

// String has useful methods — you'll use these in automation
int len    = url.length();          // 30
String up  = url.toUpperCase();     // "HTTPS://API..."
boolean has = url.contains("users"); // true
String sub = url.substring(8, 19); // "api.example"
Quick Comparisons

When to use which — in 30 seconds:

int

Whole numbers only. Smaller in memory. Faster operations.

int age = 25;
vs
double

Decimals required. More memory. Use whenever fractions matter.

double rate = 8.5;
double pi = 3.14159;
char

Exactly ONE character. Uses single quotes. It's a primitive.

char grade = 'A';
char sep = ',';
vs

Zero OR more characters. Uses double quotes. It's an object (class).

Common Variable & Type Errors

These are the most frequent mistakes when declaring variables. Know them once — avoid them forever.

🔴 Error: Incompatible Types / Type Mismatch

What happened?

You declared a variable as int but tried to assign a decimal value. Java's type system catches this before the program runs.

error: incompatible types: possible lossy conversion from double to int int age = 25.5; ^
❌ Wrong
int age = 25.5; // decimal in int! int count = "ten"; // string in int!
✅ Correct
int age = 25; // whole number ✓ double price = 25.5; // use double ✓
✅ Rule: Match the value to the declared type. Decimal? Use double. Whole number? Use int.
🔴 Error: Missing Quotes in String Assignment

What happened?

You assigned text to a String variable without wrapping it in double quotes. Java treats unquoted words as variable names — and Rahul is not a known variable.

error: cannot find symbol String name = Rahul; ^ symbol: variable Rahul location: class HelloWorld
❌ Wrong
String name = Rahul; String city = 'Mumbai'; // single quotes!
✅ Correct
String name = "Rahul"; // double quotes ✓ String city = "Mumbai"; // double quotes ✓
✅ Rule: Strings always use "double quotes". Single quotes are only for char.
🔴 Error: Case Sensitivity — Variable Not Found

What happened?

Java is case-sensitive. age, Age, and AGE are three completely different variables. Referencing one with the wrong case causes a compile error.

error: cannot find symbol System.out.println(Age); ^ symbol: variable Age location: class HelloWorld
❌ Wrong
int age = 25; System.out.println(Age); // 'Age' ≠ 'age' String Name = "Rahul"; System.out.println(name); // 'name' ≠ 'Name'
✅ Correct
int age = 25; System.out.println(age); // exact match ✓ String name = "Rahul"; System.out.println(name); // exact match ✓
✅ Rule: Always use exact same spelling + case when referencing a variable.
🔴 Error: Invalid Variable Name

What happened?

You used a name that violates Java's naming rules — like starting with a number, using a space, or using a reserved keyword as the variable name.

error: not a statement int 2count = 10; ^ error: ';' expected int 2count = 10; ^
❌ Wrong
int 2count = 10; // starts with digit int user name = 5; // has a space int int = 1; // 'int' is a keyword
✅ Correct
int count2 = 10; // digit at end ✓ int userName = 5; // camelCase ✓ int itemCount = 1; // descriptive ✓
✅ Rule: Start with a letter or _ or $. Use camelCase. Never use Java keywords.
