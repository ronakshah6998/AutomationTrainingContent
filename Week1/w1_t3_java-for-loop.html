For Loop Basics in Java

Stop writing the same code ten times. Use a loop — let Java do the repetition for you.

🔁 For Loop ⚙ Syntax Breakdown ▶ Live Simulator 🧠 Quick Check
What is a Loop?

A loop lets Java repeat a block of code multiple times — without you writing it multiple times.

💡
Real-world analogy: Imagine printing attendance for 30 students. Without a loop, you write System.out.println() 30 times. With a loop, you write it once — Java runs it 30 times automatically. That's the power of looping.
🔑
Core idea: A for loop runs as long as a condition is true. When the condition becomes false, the loop stops and execution continues normally.
Without a Loop vs With a Loop
❌ Without a Loop (tedious) 
System.out.println(1);
System.out.println(2);
System.out.println(3);
System.out.println(4);
System.out.println(5);
// 5 lines for 5 numbers
// What if you need 100?
✅ With a For Loop (clean) 
for (int i = 1; i <= 5; i++) {
    System.out.println(i);
}
// 3 lines, any number of
// iterations you want
Why Loops Matter in Automation
📋

Multiple Test Data

Loop through an array of usernames and run the same login test for each one — no code duplication.

🔄

Repeating Actions

Retry a failed API call up to 3 times. for (int i = 0; i < 3; i++) wraps the retry logic cleanly.

📊

Processing Collections

Loop through a list of API responses and assert each one — Selenium, RestAssured, and TestNG all rely on this pattern.

⏱

Wait / Poll Logic

Poll an element up to 10 times with a delay. Loop + condition + break is the foundation of custom waits.

Anatomy of a For Loop

Every for loop has exactly three parts inside the parentheses. Each part has a specific job — hover to learn each one.

for Keyword
 (
int i = 1 ① Init
; 
i <= 5 ② Condition
; 
i++ ③ Update
 ) {
... ④ Body
 }
Init: runs once
Condition: checked every iteration
Update: runs after each body
Body: what repeats
Full Example with Trace
☕ Print 1 to 5 
for (int i = 1; i <= 5; i++) {
    System.out.println("Value of i: " + i);
}

// Output:
// Value of i: 1
// Value of i: 2
// Value of i: 3
// Value of i: 4
// Value of i: 5
Step-by-Step Execution
1

Initialization — runs once before anything

int i = 1 — Java creates the variable i and sets it to 1. This only happens one time.

2

Condition Check — evaluated before each iteration

i <= 5 — If this is true, Java enters the loop body. If false, the loop ends immediately.

3

Body Executes — the repeating code

System.out.println(i) — Whatever is inside { } runs. At this point, i has its current value.

4

Update — runs after each body execution

i++ — Java increments i by 1. Then it jumps back to step 2 and checks the condition again.

5

Loop Ends — condition becomes false

When i reaches 6, the condition 6 <= 5 is false → loop exits. Code after the loop runs next.

More Loop Patterns
☕ Common Variations 
// Count down
for (int i = 5; i >= 1; i--) {
    System.out.println(i);   // 5 4 3 2 1
}

// Step by 2
for (int i = 0; i <= 10; i += 2) {
    System.out.println(i);   // 0 2 4 6 8 10
}

// Loop over an array
String[] users = {"Alice", "Bob", "Carol"};
for (int i = 0; i < users.length; i++) {
    System.out.println("User: " + users[i]);
}

 Common For Loop Mistakes

      
These four errors cover 90% of beginner loop bugs. Learn each one once.


      

        
        
        
        
      


      
      

        
🔴 Error: Infinite Loop — Missing or Broken Update

        

          
What happened?

          
The loop variable is never updated (or updated in the wrong direction), so the condition stays true forever. The program hangs — it never exits the loop.

          
// Program hangs — i never changes, condition always true
for (int i = 1; i <= 5; ) {
    System.out.println(i);  // prints "1" forever
    // i++ was forgotten!
}

          

            

              
❌ Wrong (no update)

              
for (int i = 1; i <= 5; ) {
    System.out.println(i);
    // i never increments!
}

            

            

              
✅ Correct

              
for (int i = 1; i <= 5; i++) {
    System.out.println(i);
    // i increments each time ✓
}

            

          

          
✅ Rule: Always ensure your update moves i towards making the condition false.

        

      


      
      

        
🔴 Error: Wrong Condition — Loop Never Runs

        

          
What happened?

          
The condition is already false before the first iteration begins. Java checks the condition first — if it's false from the start, the body never executes at all.

          
// Loop body NEVER runs — 1 is not greater than 5
for (int i = 1; i > 5; i++) {
    System.out.println(i);  // zero output
}

          

            

              
❌ Wrong (condition impossible)

              
for (int i = 1; i > 5; i++) {
    System.out.println(i);
    // never executes
}

            

            

              
✅ Correct

              
for (int i = 1; i <= 5; i++) {
    System.out.println(i);
    // runs 5 times ✓
}

            

          

          
✅ Rule: Make sure the initial value satisfies the condition on the very first check.

        

      


      
      

        
🔴 Error: Off-by-One — Using < vs <= (or > vs >=)

        

          
What happened?

          
Using < instead of <= causes the loop to run one fewer time than intended. This is one of the most frequent silent bugs in loop code.

          
// Intended: print 1 to 5
// Actual output: 1 2 3 4  (misses 5!)
for (int i = 1; i < 5; i++) {
    System.out.println(i);
}

          

            

              
❌ Wrong (stops at 4)

              
for (int i = 1; i < 5; i++) {
    System.out.println(i);
    // prints 1, 2, 3, 4 only
}

            

            

              
✅ Correct (includes 5)

              
for (int i = 1; i <= 5; i++) {
    System.out.println(i);
    // prints 1, 2, 3, 4, 5 ✓
}

            

          

          
Quick rule of thumb:

          

            
OperatorIncludes end value?Example (start=1, end=5)Runs

            
              
<❌ Noi < 54 times (1,2,3,4)

              
<=✓ Yesi <= 55 times (1,2,3,4,5)

            
          

          
✅ Rule: Use <= when you want to include the end value. Use < for array indices (0-based).

        

      


      
      

        
🔴 Error: Incorrect Loop Direction — Increment When Should Decrement

        

          
What happened?

          
You want to count down but used i++ instead of i--. The update moves i further from the end condition, creating an infinite loop.

          
// Intended: count down from 5 to 1
// Actual: infinite loop — i keeps going 5, 6, 7, 8...
for (int i = 5; i >= 1; i++) {  // should be i--
    System.out.println(i);
}

          

            

              
❌ Wrong (wrong direction)

              
for (int i = 5; i >= 1; i++) {
    System.out.println(i);
    // i goes 5,6,7,8... infinite!
}

            

            

              
✅ Correct (count down)

              
for (int i = 5; i >= 1; i--) {
    System.out.println(i);
    // prints 5, 4, 3, 2, 1 ✓
}

            

          

          
✅ Rule: Counting up → use i++. Counting down → use i--. Condition must match direction.
