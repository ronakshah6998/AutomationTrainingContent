Methods & Class Structure in Java

Learn how Java organises code into classes and methods — the foundation of every Java program ever written.

🏗️ Class Structure 🔧 Methods ▶ Live Simulator 🧠 Quick Check
What is a Class?

A class is the fundamental building block of every Java program — it bundles data and behaviour together in one place.

💡
Real-world analogy: A class is like a blueprint. The blueprint for a house defines how many rooms it has, what the walls look like, and what actions can happen inside. The actual house built from it is an object. In Java, you define the blueprint once (class) and create as many houses (objects) as you need.
🔧
What is a Method? A method is a named block of code inside a class that performs a specific task. Think of it as a button on a machine — press it and something defined happens. You can press the same button many times without rewriting the logic.
Class vs Method — At a Glance
ConceptWhat It IsAnalogySyntax
Class A blueprint/container for data and methods House blueprint public class Car { }
Method A named action defined inside a class Button on a machine void drive() { }
Object A real instance created from a class Actual house built Car myCar = new Car();
Why This Matters in Automation
♻️

Reusable Logic

Write a login() method once. Call it in 50 test cases. Any change to login logic is fixed in one place — not 50.

🧹

Clean Code Structure

Group related methods in one class — LoginPage, ApiHelper, TestUtils. Navigation becomes intuitive.

🔄

Page Object Model

Selenium's POM pattern is entirely built on classes and methods. Each page is a class; each action is a method.

Test Helper Methods

waitForElement(), takeScreenshot(), readExcelData() — all reusable methods you'll write and call constantly.

A Java Class — Line by Line

Every Java program has at least one class and one main method. Here is the minimal structure — and what every part means.

☕ HelloWorld.java — Full Class 
public class HelloWorld {

    public static void main(String[] args) {

        System.out.println("Hello, World!");

    }

}
Breaking It Down
public class HelloWorld { public class name
public static void main(String[] args) { entry point parameters
// your code goes here body
} closes main
} closes class
1

public — Access Modifier

Makes the class accessible from anywhere. For classes containing main, this must always be public.

2

class HelloWorld — Class Declaration

The keyword class tells Java this is a class definition. HelloWorld is the name — it must match the filename exactly (HelloWorld.java).

3

public static void main(String[] args) — Entry Point

Java looks for this exact signature to start execution. static means no object is needed; void means it returns nothing; args accepts command-line arguments.

4

{ } — Curly Braces

Opening { starts a block. Closing } ends it. Every class body and every method body must be wrapped in matching braces.

A Real-World Class Example
☕ Car.java — Class with Multiple Methods 
public class Car {

    // Fields (data stored in the class)
    String brand  = "Toyota";
    int    speed  = 0;

    // Method 1 — no return value
    public void accelerate() {
        speed += 10;
        System.out.println("Speed: " + speed);
    }

    // Method 2 — returns a value
    public String getBrand() {
        return brand;
    }

    // Entry point
    public static void main(String[] args) {
        Car myCar = new Car();  // create object
        myCar.accelerate();        // call method
        System.out.println(myCar.getBrand());
    }
}
Method Syntax — Every Part Explained

A method has five key parts. Each one has a specific role — miss any one and Java will not compile.

public  static  int  add(int a, int b) {  return a + b;  }
① access: public ② static ③ return type: int ④ name: add ⑤ parameters: (int a, int b) ⑥ body: { return a+b; }
Return Type

void / int / String

Declares what type of value the method gives back. void means it returns nothing.

Method Name

camelCase name

Identifies the method. Call it by this name. Convention: start with a verb — getName, calculateTax.

Parameters

(int a, int b)

Inputs the method receives. Each needs a type and name. Use empty () if no inputs needed.

Body + Return

{ logic; return; }

The block of code that runs. If return type is not void, must have a return statement.

void vs Return — Side by Side
☕ void — No return value 
// void = does not return anything
public static void greet(String name) {
    System.out.println("Hello, " + name);
}

// Calling it:
greet("Rahul");
// Output: Hello, Rahul
☕ int — Returns a value 
// int = returns a whole number
public static int add(int a, int b) {
    return a + b;
}

// Calling it:
int result = add(3, 7);
System.out.println(result); // 10
More Method Examples
☕ Method Patterns Used in Automation 
// 1. No parameters, no return — action method
public static void clickLoginButton() {
    driver.findElement(By.id("login")).click();
}

// 2. Parameters, no return — data input method
public static void enterCredentials(String user, String pass) {
    usernameField.sendKeys(user);
    passwordField.sendKeys(pass);
}

// 3. Parameters, returns value — utility method
public static int calculateDiscount(int price, int pct) {
    return price - (price * pct / 100);
}

// 4. No parameters, returns value — getter method
public static String getPageTitle() {
    return driver.getTitle();
}
How a Method Call Works
1

Method is called

When Java reaches add(3, 7) it stops the current code and jumps to the add method definition.

2

Arguments become parameters

3 is assigned to a, 7 is assigned to b. These are local to the method — they don't exist outside it.

3

Method body executes

return a + b → return 3 + 7 → return 10. The result is handed back to the caller.

4

Execution returns to the caller

The value 10 is placed where the method was called. int result = add(3, 7) → int result = 10.

 Common Method & Class Mistakes

      
These four errors cause most beginner Java compilation failures. Know each one — avoid them forever.


      

        
        
        
        
      


      
      

        
🔴 Error: Missing return Statement in Non-void Method

        

          
What happened?

          
If a method declares a return type (e.g. int, String), Java requires a return statement in every possible code path. Missing it causes a compile error — even if the logic seems obvious.

          
error: missing return statement
    public static int add(int a, int b) {
        int result = a + b;
        // forgot to return!
    }                       ^

          

            

              
❌ Wrong (no return)

              
public static int add(int a, int b) {
    int result = a + b;
    // result computed but not returned!
}

            

            

              
✅ Correct

              
public static int add(int a, int b) {
    int result = a + b;
    return result; // ✓ returned
}

            

          

          
✅ Rule: Every non-void method must have a return statement that returns the declared type.

        

      


      
      

        
🔴 Error: Returning the Wrong Type

        

          
What happened?

          
The method declares it returns int, but the return statement hands back a String or double. Java is strictly typed — the returned value must exactly match the declared return type.

          
error: incompatible types: String cannot be converted to int
    public static int getScore() {
        return "100";    // wrong! String ≠ int
    }               ^

          

            

              
❌ Wrong (type mismatch)

              
public static int getScore() {
    return "100"; // String ≠ int!
}

            

            

              
✅ Correct

              
public static int getScore() {
    return 100; // int ✓
}
// OR change return type to String

            

          

          
✅ Rule: The value after return must match exactly what the method signature declares.

        

      


      
      

        
🔴 Error: Incorrect Method Call — Wrong Number of Arguments

        

          
What happened?

          
The method definition says it needs two int parameters, but the call passes one, three, or the wrong types. Java checks this at compile time and rejects it immediately.

          
error: method add in class Main cannot be applied to given types
    int result = add(5);    // expected 2 args, got 1
                 ^
required: int, int
found:    int

          

            

              
❌ Wrong (wrong args)

              
public static int add(int a, int b) {...}

// Calls with wrong count:
add(5);          // 1 arg → error
add(1,2,3);    // 3 args → error

            

            

              
✅ Correct

              
public static int add(int a, int b) {...}

// Exactly 2 ints:
int r = add(3, 7); // ✓

            

          

          
✅ Rule: The number and types of arguments in the call must exactly match the method's parameter list.

        

      


      
      

        
🔴 Error: Missing or Mismatched Curly Braces

        

          
What happened?

          
Every opening { must have a matching closing }. A missing brace causes cascading errors — Java loses track of where the method and class boundaries are, giving confusing messages.

          
error: reached end of file while parsing
}
^
1 error   (missing closing brace for method or class)

          

            

              
❌ Wrong (missing brace)

              
public class Main {
    public static void main(String[] args) {
        System.out.println("Hi");
    // missing } to close main!
} // this closes class — not main

            

            

              
✅ Correct

              
public class Main {
    public static void main(String[] args) {
        System.out.println("Hi");
    }  // closes main ✓
}  // closes class ✓

            

          

          
✅ Rule: For every { there must be a matching }. Count them — they must be equal.
